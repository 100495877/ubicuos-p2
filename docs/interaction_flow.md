# Flujos de interacción

## Registro de gasto
1. Usuario realiza un gesto con el controlador
2. El sistema entra en modo escucha
3. Usuario indica el gasto (voz o simulación)
4. El gasto se envía al servidor
5. La pantalla muestra el gasto
6. Usuario confirma

## Evaluación de gasto (Tinder)
1. La pantalla muestra un gasto
2. Usuario decide:
   - Me gusta
   - No me gusta
3. El sistema registra la decisión
4. Se genera feedback visual/sonoro

## Recap mensual
1. El sistema agrupa gastos
2. Muestra métricas:
   - total
   - categorías
   - gastos repetidos
   - gastos no deseados
